# LLD: PWA Integration for AegisCore

## 1. Overview
This document specifies the implementation of Progressive Web App (PWA) features for the AegisCore web interface. The integration will use `vite-plugin-pwa` for automated service worker generation and manifest management.

## 2. Technical Details

### 2.1 Frontend Configuration
- **Package**: `vite-plugin-pwa`
- **Vite Integration**: Add `VitePWA` to the `plugins` array in `vite.config.ts`.
- **Manifest Properties**:
  - `name`: AegisCore
  - `short_name`: Aegis
  - `theme_color`: #1a1a1a (Deep dark theme to match AegisCore branding)
  - `background_color`: #1a1a1a
  - `display`: standalone
  - `icons`:
    - `pwa-192x192.svg` (Type: image/svg+xml, sizes: 192x192)
    - `pwa-512x512.svg` (Type: image/svg+xml, sizes: 512x512)

### 2.2 Asset Handling
Since the frontend uses `rust_embed` in the backend, all files generated in `frontend/dist/` must be served correctly. The `manifest.webmanifest` file generated by Vite requires a specific MIME type.

### 2.3 Backend Changes
The `aegis-web` crate needs to recognize the `.webmanifest` extension in its `mime_for_path` utility.

## 3. Implementation Plan

### Milestone 38: PWA Integration
- **Task 38.1**: Frontend PWA setup (Vite plugin, manifest, icons).
- **Task 38.2**: Backend MIME type support for `.webmanifest`.
